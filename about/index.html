關於我

👋 嗨！歡迎來到我的技術園地

我是一位熱愛學習與分享的全端軟體工程師，專注於 Web 全端開發、雲端技術與 AI 技術應用。擁有豐富的前後端開發經驗，熱衷於探索新技術並實踐於專案開發中。

💭 為什麼建立這個部落格？

相信許多開發者都有類似的經驗：

“嗯…這個 Bug 好像以前解決過，但我只大概記得當時怎麼處理，或只記得處理的片段。“

於是又要重新 Google、重新翻文件、重新踩一次坑。這種無限輪迴的挫折感，讓我決定開始認真記錄自己的學習歷程。

這個部落格不只是我的技術筆記本，更是：

  • 📝 記錄曾經跌過的坑，避免重複踩雷
  • 🛠️ 整理學習的工具與知識，建立自己的知識體系
  • 📚 以教學為導向的文章，在分享中加深理解
  • 🌱 持續成長的証明，見証技術能力的累積

🔧 技術領域

目前主要關注的技術方向包括：

Web 全端開發

前端技術

  • 核心技術：HTML5、CSS3、JavaScript、TypeScript
  • 框架/函式庫：React、jQuery、Bootstrap
  • 開發方式：響應式設計、AJAX、結構化程式設計

後端技術

  • 主要語言：C#、Python、Solidity
  • 框架：.NET Core、ASP.NET、.NET MVC
  • API 設計：RESTful API、SignalR
  • 訊息佇列：RabbitMQ
  • 任務排程：HangFire

資料庫

  • RDBMS：MSSQL、SSMS
  • 報表服務：SSRS
  • 資料庫設計：資料庫程式設計、效能優化、索引調校

伺服器與部署

  • Web 伺服器：IIS
  • 作業系統：Windows Server、Linux

雲端與容器化技術

Google Cloud Platform (GCP)

  • 服務：相關GCP生態系服務
  • 分析工具：GA4、GTM、Looker Studio
  • 資料庫：Metabase

Microsoft Azure

  • 雲端服務：Azure 相關服務與部署

容器化與編排

  • Docker：容器化應用程式部署
  • Microservices：微服務架構設計與實作

軟體工程實踐

版本控制

  • 平台：GitHub、Gitea
  • 工作流程：Git Flow、CI/CD

開發方法論

  • 敏捷開發：Agile
  • 測試驅動：TDD (Test-Driven Development)、BDD (Behavior-Driven Development)
  • 規格驅動：SDD (Specification-Driven Development)
  • 設計模式：Design Patterns

DevOps

  • 持續整合/部署：CI/CD Pipeline
  • 日誌與監控：ELK Stack
  • 資訊安全：ISO27001

AI 開發工具

AI 輔助開發

  • GitHub Copilot：程式碼自動完成與建議
  • Claude Code：AI 程式設計助手
  • Codex：OpenAI 程式碼生成模型

AI Agent 與應用

  • 對話模型：Gemini、Claude
  • AI Agent：Agent Skill、MCP (Model Context Protocol)
  • CLI 工具：X CLI
  • 技術研究：OpenClaw、AutoGPT、Prompt Engineering、RAG 技術

開發工具鏈

  • IDE：Visual Studio、VS Code
  • 部落格框架：Hexo、GitHub Pages
  • 其他工具：Postman、各類開發輔助工具

📖 部落格內容方向

這裡你可以找到：

  1. 教學文章：從零開始的完整教學，適合初學者跟隨學習
  2. 踩坑日常：實戰中遇到的問題與解決方案
  3. 技術筆記：整理的技術知識點與最佳實踐
  4. 工具分享：好用的開發工具與使用技巧

🎯 我的目標

  • 💪 持續精進技術能力，深化全端開發與雲端架構設計能力
  • 🏗️ 掌握現代化開發實踐，包含微服務架構、DevOps 與 CI/CD
  • 🤖 探索 AI 輔助開發，提升開發效率與程式碼品質
  • 📝 產出高品質教學文章，幫助更多學習者少走彎路
  • 🤝 加入優秀的開發團隊，與志同道合的夥伴一同成長
  • 🌐 建立技術社群連結，在分享與交流中持續進步
